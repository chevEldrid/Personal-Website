Snowfall

So last week I mentioned I wanted to go into the Snowfall decision engine a little bit and explain the process used to parse user input. This idea has been with me from back when Julian and I designed Harry Potter: Messed-Up Memories, a very clumsy but fun take on old school text adventures back in High School. Since then, my code has become far more sophisticated but I managed to keep the same design philosphy. Is this the best way to do it? I'm not sure, but so far I have not found a decidedly better way.

Depending on player position in-game, a different set of available actions are assigned to the commands ArrayList and it's this list that is checked every time a player submits a command. First, the user input is split into an array by spaces. Then, the game runs the first word of input against the command ArrayList looking for matches. If there is one, great! We've figured out what the user wants! The rest of the input is then stored as potential arguments (ex: "inpsect" is the command and the user said "inspect lighter" so "lighter" becomes an argument passed to the "inpsect" command) If a command is not recognized in the first word, the command ArrayList will then compare to the first two words of user input. This continues until all input is checked and finally if nothing matches, the game returns an error. This way, commands of any number of words or potential arguments can be used with little hassle!
